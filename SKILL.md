# Physics Lab Report Skill

当任务是“根据实验材料写大学物理实验报告”时使用本 skill，而不是凭记忆直接生成内容。

本 skill 采用“仓库内置课程资料 + 用户本地个性化材料”的方式工作：

- 仓库内置：
  - `course-materials/绪论课ppt.pdf`
  - `course-materials/01_课程基础知识_测量误差及数据处理的基础知识.pdf`
  - `course-materials/02_课程基础知识_电磁学实验基本仪器.pdf`
  - `course-materials/03_课程基础知识_光学实验预备知识.pdf`
  - `course-materials/04` 到 `10` 的 7 个主题实验 PDF
- 用户本地补充：
  - 实验数据记录表照片
  - 老师下发的对应主题实验 PPT
  - 用户已有的 LaTeX 模板、脚本、表格等

## 核心规则

- 绝不编造实验数据。
- 对模糊 OCR 结果绝不自行“修正”。
- 只要课程资料已经规定报告格式，就不得擅自另起一套结构。
- 在用户确认提取出的数据之前，不得进入数据分析和正文撰写。
- `course-materials/绪论课ppt.pdf` 是实验报告要求的首要依据，尤其是不确定度、有效位数、作图和线性回归写法。
- 如果老师补充 PPT 与实验指导 PDF 在实验步骤、仪器常数或记录格式上有更新，优先采用老师补充材料；但报告规范仍应尽量符合 `绪论课ppt.pdf` 的相关要求。
- 如果本地存在合适的 LaTeX 编译工具链，应优先在本地实际编译，生成可打开的 PDF，并根据编译结果修正工程问题。
- 如果本地不存在合适的 LaTeX 编译工具链，仍必须保证生成的报告工程在用户把整个文件夹上传到 Overleaf 后可以直接编译成功。
- 报告中使用的图片必须放在项目内部的相对路径下，不能引用用户本机绝对路径。

## 材料使用顺序

默认按下面顺序取材：

1. 先读 `course-materials/绪论课ppt.pdf`，明确实验报告格式、有效位数、不确定度、作图和线性回归要求。
2. 再读当前实验对应的主题实验 PDF。
3. 如有必要，再读相关课程基础知识 PDF。
4. 再读用户本地的老师补充 PPT、讲义或说明。
5. 最后处理用户本地的实验数据记录表照片、扫描件、表格或脚本。

优先从 [references/bundled-course-materials.md](./references/bundled-course-materials.md) 获取内置材料索引；需要系统搜索本地额外文件时，再读 [references/material-discovery.md](./references/material-discovery.md)。
涉及图片裁剪、插图整理和 Overleaf 可编译性时，再读 [references/figures-and-overleaf.md](./references/figures-and-overleaf.md)。

## Subagent 使用策略

如果当前代理环境支持 subagent，应尽量把“边界清楚、输出可结构化回传、与最终成文低耦合”的任务拆出去做，以减少单个 agent 的上下文膨胀。

推荐优先拆分的任务：

- 课程规范提取：读取 `course-materials/绪论课ppt.pdf`，只提取报告结构、不确定度、有效位数、线性回归和作图要求。
- 主题实验解读：读取当前实验对应的主题实验 PDF，提取实验目的、原理、关键公式、步骤、记录方式、误差来源、思考题，以及需要完成的作图和拟合任务。
- 基础知识补读：按实验类型分别补读 `01`、`02`、`03` 基础知识 PDF，并只返回与当前实验直接相关的结论。
- 老师补充材料定位与提要：在工作区内搜索老师 PPT、PDF、讲义或说明，并总结其中对实验步骤、仪器常数、记录格式、讨论要求的更新。
- 原始数据双重转录：对同一份数据照片做两次相互独立的识别。若环境允许，优先使用两个独立 subagent；主 agent 只负责比对冲突并整理给用户确认。
- 本地脚本与模板复用审查：检查已有 `tex`、`py`、`csv` 是否属于同一实验、公式是否一致、是否可直接复用。
- 图片整理方案：判断哪些图像应直接使用原图，哪些需要从原始照片中裁剪，并给出建议文件名与 `figures/` 目录落点。
- 数据处理与绘图子任务：将线性回归、中间数据表生成、不确定度传播、派生量计算、绘图脚本编写拆为独立子任务。
- 编译与交付巡检：若本地有可用 LaTeX 工具链，则独立执行一次本地编译并检查 PDF 是否产出；若没有，则最终独立检查相对路径、图片文件存在性、宏包兼容性、是否依赖本地字体、是否依赖 shell-escape 或编译时运行脚本，并以 Overleaf 可直接编译为最低交付标准。

推荐的主从分工：

- 主 agent 负责确定当前实验、选择最小阅读集、协调任务顺序、汇总结论、发现冲突并向用户提问。
- subagent 负责读取单一材料、处理单一图像集、完成单一计算模块，返回结构化摘要、候选结论、疑点列表或局部工件。
- 主 agent 在汇总 subagent 结果时，应尽量保留摘要和结论，而不是把整段原文继续带入后续上下文。

推荐并行顺序：

1. 主 agent 先确定实验名称、报告类型候选和最小阅读集。
2. 并行启动课程规范提取、主题实验解读、老师补充材料定位、原始数据第一次转录、原始数据第二次转录。
3. 主 agent 汇总并比较结果，整理出 `data.tex` 草稿和待确认疑点，交给用户确认。
4. 用户确认数据后，再并行启动数据处理脚本、绘图脚本、图片裁剪整理、旧脚本复用审查。
5. 主 agent 最后统一写 `report.tex`，并安排一次编译与交付巡检：本地可编译时做实际编译，否则做 Overleaf 兼容性巡检。

以下任务不应下放给 subagent 直接拍板：

- 用户确认关口，尤其是 `data.tex` 的最终确认。
- 多份材料冲突时的最终取舍，例如老师 PPT 与主题实验 PDF 在步骤或常数上的冲突。
- 全文统一修约、统一术语、统一 LaTeX 结构与引用路径。
- 报告最终成稿与交付前检查。

## 必须执行的流程

### 1. 先完整理解实验

在动笔前，必须从主题实验 PDF 与老师 PPT 中提取并理解：

- 实验背景、目的与研究对象
- 实验原理、关键公式、推导思路
- 仪器装置、实验接线或光路/结构
- 实验步骤与数据记录方式
- 注意事项、误差来源、思考题
- 实验需要画什么图、做什么拟合、算什么物理量

如果主题实验明显依赖基础知识，必须一并补读。例如：

- 光学类实验通常需要参考 `03_课程基础知识_光学实验预备知识.pdf`
- 电磁学/仪器类实验通常需要参考 `02_课程基础知识_电磁学实验基本仪器.pdf`
- 凡涉及不确定度、数据处理、有效位数时，都应参考 `01_课程基础知识_测量误差及数据处理的基础知识.pdf`

### 2. 确定报告类型

根据 `绪论课ppt.pdf`、实验指导书和老师要求，判断本次应写：

- 完整实验报告
- 简要实验报告
- 或老师单独要求的其他格式

不要把某个实验名硬编码为某种报告类型。若材料没有明确说明，再询问用户。

### 3. 先识别并核对原始数据

如果存在实验数据记录表照片或扫描件，必须先提取数据。

推荐做法：

- 做两次相互独立的数据识别。
- 如果当前环境和策略允许调用 subagent，就用两个独立 subagent 分别识别。
- 如果不允许，就用两种不同的本地识别策略做双重转录。

逐项对照以下内容：

- 表头
- 单位
- 正负号
- 小数点
- 多次测量值
- 平均值
- 仪器设置或常数
- 手写补注

遇到以下情况必须停止并找用户确认：

- 数字无法辨认
- 两次识别结果冲突
- 缺少会影响计算的单位
- 值明显可疑，但无法从原始图片中证实

确认后的数据要单独写入一个 LaTeX 文件，通常命名为 `data.tex`，并尽量保持原始记录表的列结构和书写习惯。

写完 `data.tex` 后，必须把识别结果展示给用户，请用户确认“数据识别是否正确”。未经用户确认，不得进入下一步。

### 3.5 处理实验相关图片

实验数据中可能含有以下两类图像：

- 已经单独存在的图像文件，例如波形图、电路图、装置图、光路图
- 嵌在原始照片中的局部图像，例如数据记录表照片中的示波器波形、纸面电路图、手绘光路图

处理规则：

- 如果图像本来就是单独文件，优先直接使用原图。
- 如果图像嵌在原始照片中，应裁剪出对应区域，另存为单独文件后再插入报告。
- 裁剪时只允许做有助于辨认和排版的处理，例如旋转、裁边、留出适当边距、必要的方向校正。
- 不得通过修图改变图像内容本身，不得添加不存在的线条、文字或波形。
- 如果一个原始照片里既有数据表又有波形图/电路图，数据识别与图像裁剪应分开处理。

生成图片时，建议统一放入项目内的 `figures/` 目录，使用稳定、简短、ASCII 文件名，例如：

- `figures/waveform-1.png`
- `figures/circuit-setup.png`
- `figures/raw-table-photo-1.jpg`
- `figures/waveform-1-crop.png`

### 4. 数据处理与计算

需要计算时，优先写代码而不是手算堆在正文里。常见情况包括：

- 线性回归
- 中间数据表生成
- 不确定度传播
- 可重复使用的派生量计算
- 绘图

如果工作区已有可复用的脚本，先核对其是否属于同一个实验、公式是否一致，再决定是否复用。

线性回归图要求：

- 只保留散点和回归线
- 不要保留散点两两之间的折线
- 坐标轴名称与单位清晰
- 图中或正文中明确给出拟合方程
- 相关系数 `r` 的保留规则要符合 `绪论课ppt.pdf`；如果课程要求是“保留到非 9 的后三位”，就按此执行

在最终确定不确定度、有效位数、回归结果保留位数之前，必须再检查 [references/numerical-rules.md](./references/numerical-rules.md)，并以 `绪论课ppt.pdf` 为准。

### 5. 写 LaTeX 实验报告

中文报告优先使用适合中文环境的 LaTeX 方案，例如 `ctexart`。

建议文件组织：

- 主文件：`report.tex` 或实验名对应的 `.tex`
- 数据表：`data.tex`
- 图像目录：`figures/`
- 图像：`figures/*.png`、`figures/*.jpg` 或 `figures/*.pdf`
- 分析脚本：`*.py`

核心计算过程应尽量写清：

- 物理量定义
- 代入公式
- 代入数值
- 最终结果和单位

实验核心结果不要只给结论而省略关键代入步骤。

如果课程要求附原始签字数据或记录表，应将原图放入附录或作为附件保留。

## 本地编译与交付要求

生成最终工程时，必须优先判断当前环境是否存在合适的 LaTeX 编译工具链，例如可用于中文文档的 `latexmk`、`xelatex` 或等效方案。

处理规则：

- 如果本地工具链可用，应实际运行一次本地编译，拿到 PDF 结果，并修正编译报错、缺图、路径错误、缺失输入文件等问题。
- 如果本地工具链不可用，不要伪称“已经编译通过”；应改为执行静态检查，并保证该工程上传到 Overleaf 后可以直接编译。
- 无论是否本地编译，最终都不得要求用户在编译前手动改路径、改宏包、改字体或先运行本地脚本生成中间文件。

## Overleaf 兼容性要求

生成最终工程时，必须保证用户将整个项目文件夹上传到 Overleaf 后可以直接编译。至少满足：

- 主文件、`data.tex`、图像文件都在同一项目目录树内
- 所有 `\input{}` 和 `\includegraphics{}` 都使用相对路径
- 不引用本机绝对路径
- 不依赖本地安装字体
- 不依赖 shell-escape
- 不依赖用户先在本地运行 Python、MATLAB 或其他脚本来生成 LaTeX 编译时必需的文件
- 如果使用中文文档类或中文宏包，选择 Overleaf 常见 TeX Live 环境可直接处理的写法
- 如果用了 Python 做线性回归，应把生成好的图像文件保存进项目，而不是在 Overleaf 编译时现场跑 Python
- 尽量使用 Overleaf 默认可用的常见宏包与 `ctexart`

如果需要把原始数据照片、裁剪图、回归图都插入报告，应确保这些文件也一并保存在项目内，并且文件名与引用一致。

## 默认数值规则

以下只是默认值；一旦 `绪论课ppt.pdf` 或老师要求给出更具体规则，必须服从课程要求：

- 最终结果一般写为 `Y = y \pm \Delta`，并带单位
- `y` 的最后一位应与 `\Delta` 的最后一位对齐
- 不确定度通常保留 1 到 2 位有效数字
- 中间计算多保留几位，最后统一修约
- 需要传播不确定度时，应显式写出传播过程
- 线性回归的斜率、截距、相关系数保留位数，应以能够支持结论且符合课程要求为准

## 可用模板

如果用户没有给定自己的模板，可以从以下文件起步：

- [assets/full-report-template.tex](./assets/full-report-template.tex)
- [assets/brief-report-template.tex](./assets/brief-report-template.tex)
- [assets/data-template.tex](./assets/data-template.tex)

模板只能作为骨架，必须按实际实验与课程要求填充，不得留下未替换占位符。

## 收尾检查

结束前必须确认：

- 选择的主题实验 PDF 与用户要求的实验一致
- 相关基础知识 PDF 已在需要时阅读
- `绪论课ppt.pdf` 已被用作格式与数值规则依据
- 数据识别结果已获得用户确认
- 单位、正负号、数量级一致
- 不确定度和有效位数处理符合课程要求
- 线性回归图符合课程规范
- 单独图片与裁剪图片都已放入项目内相对路径
- 上传到 Overleaf 后不需要额外补文件或改路径
- 报告结构符合该实验要求
- 任何结论都不是建立在猜测数据上

如果还缺信息，可以停下来问用户；缺数据比编数据更可接受。
